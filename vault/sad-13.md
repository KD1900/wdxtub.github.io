上节课我们非常简略地了解了分析架构的大概方法，这节课我们再来看看如何去实现一个架构。

<!-- more -->

---

需要理解的概念主要有：

+ 实现是一个映射的过程
    + 组件、连接器、接口、配置、设计思路
    + 动态特性，非功能属性
+ 架构实现的框架
+ 评价框架
    + 平台支持：编程语言、操作系统
    + 效率
    + 其他：大小、花费、易用性、可靠性、鲁棒性、是否开源、是否便携、是否易于长期维护 
+ 中间件、架构和组件模型的关系
    + CORBA, COM/DCOM, JavaBeans, .NET, JMS, etc
+ 搭建新框架
    + 理解问题本身
    + 让架构符合问题
    + 选择框架范围
    + 避免过度工程
+ 并行和通用化技术
+ 保证架构到实现的一致性
